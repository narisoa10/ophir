# Reference

## Назначение

`docs/reference/` хранит материалы, которые помогают принимать согласованные
продуктовые, UI и UX решения. Это не источник архитектурных правил; правила
фиксируются в `docs/architecture/` и `docs/decisions/`.

## Что сюда помещается

- дизайн-референсы;
- UX-референсы;
- лучшие практики;
- эталонные экраны;
- изображения;
- screenshots текущих и целевых состояний;
- исследования интерфейсов финансовых продуктов.

## Структура

- `ui/` - визуальные референсы, компоненты, layout examples.
- `ux/` - user flows, interaction patterns, behavior references.
- `screenshots/` - эталонные или исторические screenshots.

## Как использовать

Reference помогает объяснить желаемое качество решения, но не заменяет Theme,
Typography, Color Standard или ADR. Если reference приводит к новому правилу,
это правило должно быть перенесено в architecture document.

## Запрещено

- Принимать screenshot как архитектурное решение.
- Копировать чужой UI без адаптации к Ophir.
- Хранить здесь правила, которые должны быть в `architecture/`.
