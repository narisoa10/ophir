# Spacing Standard v1

## Назначение

Spacing Standard задает единый ритм интерфейса. Отступы должны помогать
сканированию финансовых данных, разделять смысловые группы и не создавать
визуальный шум.

Этот документ является единственным источником истины для spacing roles.
Theme предоставляет доступ к spacing tokens, но не определяет их смысл отдельно
от этого стандарта.

## Tokens

### xs

Минимальный отступ между тесно связанными элементами.

Используется для: icon + label, compact metadata, small inline groups.

### sm

Малый отступ внутри компактных компонентов.

Используется для: расстояния между строками metadata, padding небольших
controls, gap между related labels.

### md

Базовый отступ системы.

Используется для: стандартный padding элементов, расстояние между row content,
разделение полей формы.

### lg

Крупный отступ между смысловыми блоками.

Используется для: разделение sections, расстояние между summary и list,
вертикальные группы на экране.

### xl

Максимальный стандартный отступ для крупных layout boundaries.

Используется для: screen-level spacing, empty states, разделение крупных зон.

## Как использовать

Spacing выбирается по смысловой дистанции:

- элементы одной фразы или control group: `xs`;
- связанные элементы внутри компонента: `sm`;
- стандартная структура компонента: `md`;
- разные блоки одного экрана: `lg`;
- крупные зоны или состояния: `xl`.

## Когда создавать новый spacing token

Новый token создается только если:

- существующие размеры не покрывают повторяющийся layout case;
- значение нужно централизованно менять;
- оно используется в нескольких компонентах или экранах;
- есть понятная роль, отличная от `xs-sm-md-lg-xl`.

## Как расширять

Перед добавлением token нужно описать:

- роль;
- где используется;
- почему существующие tokens не подходят;
- какие компоненты должны перейти на новый token.

## Запрещено

- Использовать `EdgeInsets(...)` в feature-коде.
- Добавлять spacing только для пиксельного совпадения одного экрана.
- Называть token по числу.
- Использовать большой отступ для компенсации плохой иерархии.

## Текущее ограничение

Если в существующих экранах есть raw `EdgeInsets` или числовые gaps, они
считаются миграционным долгом. Новые изменения должны либо использовать
существующие spacing tokens, либо сначала расширить этот стандарт.
