# Color Standard v1

## Назначение

Color Standard описывает палитру Ophir и правила использования цветов. Цвет
должен передавать роль: бренд, фон, поверхность, состояние или семантический
сигнал.

Этот документ является единственным источником истины для color roles. Theme
хранит конкретные color token values, но не создает новые роли без изменения
этого стандарта.

## Палитра

### Primary

Основной брендовый цвет и главный interactive акцент.

Используется для: primary actions, selected states, ключевые акценты, активные
navigation states.

### Secondary

Поддерживающий акцент.

Используется для: secondary actions, мягкие выделения, дополнительные элементы
визуальной иерархии.

### Surface

Цвет поверхностей, расположенных над background.

Используется для: cards, sheets, inputs, rows, panels.

### Background

Базовый фон приложения.

Используется для: screen background и больших layout areas.

### Semantic colors

Семантические цвета показывают состояние или финансовый смысл. Они не должны
использоваться как декоративные акценты.

#### Success

Положительное состояние.

Используется для: successful actions, positive confirmation, positive financial
signals when appropriate.

#### Warning

Состояние, требующее внимания.

Используется для: risk, incomplete data, approaching limits, caution states.

#### Error

Ошибка или destructive state.

Используется для: validation errors, failed operations, destructive actions.

#### Info

Нейтральная информационная подсказка.

Используется для: informative states, hints, neutral notices.

## Как использовать

1. Определить роль цвета.
2. Выбрать token по роли, а не по оттенку.
3. Проверить contrast и доступность.
4. Не использовать semantic color без соответствующего смысла.

## Когда можно создавать новый цвет

Новый цвет разрешен, если:

- появляется новая стабильная semantic role;
- существующая palette не выражает нужное состояние;
- цвет будет использоваться системно;
- решение описано в документации до изменения кода.

## Как расширять

Расширение палитры требует обновления этого документа, Theme и, при изменении
архитектурного принципа, ADR.

## Запрещено

- Использовать raw hex или `Color(...)` в feature-коде.
- Создавать цвет под один экран.
- Использовать Success только потому, что зеленый визуально нравится.
- Использовать Error для обычного отрицательного денежного значения без
  продуктового смысла ошибки.

## Текущее ограничение

Raw `Color(...)` внутри централизованного theme layer допустим как реализация
token values. Raw color в feature-коде считается legacy state и должен быть
заменен при ближайшем изменении соответствующего UI.
