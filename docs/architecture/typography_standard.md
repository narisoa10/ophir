# Typography Standard v1

## Назначение

Типографика Ophir строится по ролям, а не по экранам. Роль описывает функцию
текста в интерфейсе: заголовок экрана, подпись, сумма, кнопка или основной
текст. Это сохраняет визуальную иерархию при росте приложения.

Этот документ является единственным источником истины для typography roles.
`theme.md` определяет, что роли должны быть доступны через Theme, но не
переопределяет их назначение.

## Главное правило

Нельзя создавать типографику под конкретный экран. Если текст выполняет уже
известную функцию, используется существующая роль.

## Роли

### screenTitle

Назначение: главный заголовок экрана.

Где используется: Operations, Dashboard, Statistics, Budget, Settings и будущие
top-level экраны.

Примеры: "Operations", "Dashboard", "Budget".

### sectionTitle

Назначение: заголовок смыслового блока внутри экрана.

Где используется: блоки списков, summary sections, группы настроек, карточки
аналитики.

Примеры: "Recent operations", "Cash flow", "Categories".

### body

Назначение: основной читаемый текст.

Где используется: названия операций, описания, обычные значения в списках,
поясняющий текст.

Примеры: название transaction row, описание empty state.

### bodyStrong

Назначение: усиленный основной текст без превращения его в заголовок.

Где используется: важные значения строки, выбранные элементы, primary label в
compact UI.

Примеры: выбранная категория, основной label операции.

### caption

Назначение: вторичная информация.

Где используется: даты, подписи, auxiliary labels, metadata.

Примеры: "Today", "Archived", "Synced".

### captionStrong

Назначение: вторичная информация с акцентом.

Где используется: status labels, small badges, важные metadata.

Примеры: "Income", "Expense", "Pending".

### currency

Назначение: денежные значения обычного веса.

Где используется: amounts в списках, secondary totals, preview values.

Примеры: "$12.40", "-$8.99".

### currencyStrong

Назначение: ключевые денежные значения.

Где используется: balances, totals, running balance, summary cards.

Примеры: текущий баланс, total income, total expense.

### button

Назначение: текст интерактивных команд.

Где используется: primary buttons, secondary buttons, compact action buttons.

Примеры: "Save", "Add operation", "Restore".

## Как использовать

1. Определить функцию текста.
2. Выбрать роль по функции, а не по желаемому размеру.
3. Использовать typography token из Theme.
4. Если роль отсутствует, описать новую роль в этом документе до изменения кода.

## Как расширять

Новая роль добавляется только если существующие роли не выражают функцию текста.
Добавление требует описания назначения, мест использования и примеров.

## Запрещено

- Использовать `fontSize` и `FontWeight` в feature-коде.
- Создавать `dashboardTitle`, `operationAmountText` и другие screen-specific
  роли.
- Использовать `currencyStrong` для неденежных значений.
- Менять роль ради локального визуального эффекта.

## Текущее ограничение

Если существующий экран еще использует локальный text style или raw typography
value, это считается legacy state. При следующем изменении такого экрана стиль
должен быть приведен к одной из ролей этого стандарта или стандарт должен быть
обновлен до изменения кода.
